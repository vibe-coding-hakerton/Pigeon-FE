## 1. 메타 목표 (진짜 목표)

이 해커톤의 **본질적인 목표**는 Pigeon 서비스 자체가 아니라, **AI 주도 개발 파이프라인을 경험하는 것**이다.

### 1.1 핵심 경험 목표

| 목표 | 설명 |
|------|------|
| **AI 오케스트레이션** | Claude Code가 기획 → 설계 → 개발 → 테스트 → 배포까지 주도적으로 판단하고 실행 |
| **멀티 에이전트 협업** | Claude Squad를 활용한 백엔드/프론트엔드/AI 역할 분담 경험 |
| **자동화 파이프라인** | 이슈 생성 → 개발 → 커밋 → 푸시가 자동으로 이루어지는 워크플로우 구축 |
| **자율 판단 시스템** | DoD(Definition of Done), AEC(Acceptance Criteria)를 기반으로 Claude가 스스로 완료 여부 판단 |

### 1.2 활용 도구

| 도구 | 역할 |
|------|------|
| **Claude Code** | 메인 오케스트레이터 - 전체 개발 사이클 판단 및 실행 |
| **Claude Squad** | 멀티 에이전트 - 백/프론트/AI 파이프라인 역할 분담 |
| **Playwright MCP** | 프론트엔드 자동 테스트 및 오류 감지/수정 |
| **Jira MCP** | 이슈 자동 생성, 상태 관리, 진행 트래킹 |
| **LLM API** | 메일 분류 핵심 로직 (Gemini 2.5 Flash - 무료 tier 활용) |

## 2. 프로젝트 목표 (Mail-Architect)

AI 주도 개발의 **실습 대상**으로서의 프로젝트 정의.

### 2.1 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **이름** | Pigeon (AI 메일 폴더링 시스템) |
| **목적** | LLM을 활용해 이메일을 자동으로 다계층 폴더에 분류 |
| **핵심 가치** | 파일 관리 시스템 같은 직관적인 이메일 분류 UX |
| **MVP 범위** | 메일 자동 분류 + 다계층 폴더 생성 (그 외 기능 제외) |

### 2.2 핵심 기능 (MVP)

1. **메일 입력**: Gmail API 연동으로 실제 메일 동기화
2. **AI 분류**: Gemini API로 메일 내용 분석 → 적절한 폴더 경로 결정
3. **폴더 생성**: 새로운 경로일 경우 자동으로 폴더 구조 생성
4. **UI 표시**: 트리 구조로 폴더 & 메일 목록 시각화

### 2.3 기술 스택 (확정)

| 영역 | 기술 | 비고 |
|------|------|------|
| **Backend** | Django (Python) | API 서버, LLM 연동, ORM |
| **Frontend** | Next.js + TypeScript | SSR 지원, 트리 구조 UI |
| **Database** | PostgreSQL | JSONB, ArrayField, Django 통합 |
| **AI/LLM** | Gemini 2.5 Flash + LangChain | 메일 분류 로직, 모델 교체 용이 |
| **상태관리** | Zustand | 간단하고 가벼움 |
| **스타일링** | Tailwind CSS | 빠른 UI 개발 |

## 4. 성공 기준

### 4.1 메타 목표 성공 기준

- [ ] Claude Code가 Jira 이슈를 자동으로 픽업하고 개발할 수 있었다
- [ ] 코드 작성 → 테스트 → 커밋 → 푸시가 자동화되었다
- [ ] DoD/AEC 기반으로 Claude가 완료 여부를 판단했다
- [ ] Playwright로 프론트엔드 자동 테스트가 동작했다
- [ ] AI 주도 개발 워크플로우의 가능성과 한계를 파악했다

### 4.2 프로젝트 성공 기준

- [ ] 메일을 입력하면 AI가 폴더 경로를 결정한다
- [ ] 폴더 구조가 트리 형태로 UI에 표시된다
- [ ] 최소 10개 이상의 샘플 메일이 자동 분류된다
- [ ] 새로운 카테고리의 메일도 적절히 분류된다
